# 모니터 개념 정리

## Source Info
- **출처**: 자체 정리 자료
- **주제**: 동시성 프로그래밍의 모니터(Monitor) 개념
- **핵심 인물**: Tony Hoare(1974), Per Brinch Hansen(1973)

## Summary

동시성 프로그래밍에서 상호 배제(Mutual Exclusion)와 조건 동기화(Condition Synchronization)는 핵심 난제다. 이전에는 [[concepts/semaphore|세마포어]]로 해결했으나, 저수준 프리미티브인 세마포어는 `acquire()`/`release()` 순서를 잘못 쓰면 데드락이나 레이스 컨디션이 발생하는 한계가 있다.

[[concepts/monitor|모니터]]는 **공유 데이터 + 프로시저 + 조건 변수**를 하나의 캡슐로 묶은 동기화 추상화다. 핵심 규칙은 모니터 내부에 한 시점에 최대 하나의 스레드만 실행될 수 있다는 것이며, 이를 컴파일러/런타임이 자동으로 보장한다. 프로그래머는 락을 직접 관리할 필요 없이, 조건 변수의 `wait()`과 `signal()`로 "언제 기다릴지"와 "언제 깨울지"만 명시하면 된다.

모니터에는 두 가지 주요 의미론이 있다. **Hoare Monitor(Signal-and-Wait)**는 `signal()` 호출자가 즉시 모니터를 양보하여, 깨어난 스레드가 조건 보장을 받는다(`if`로 충분). **Mesa Monitor(Signal-and-Continue)**는 `signal()` 호출자가 계속 실행하며, 깨어난 스레드는 나중에 실행되므로 조건을 재확인해야 한다(`while` 필수). Mesa는 컨텍스트 스위치가 적고 구현이 단순하여, Java `synchronized`/`wait()`/`notify()`, C# Monitor, POSIX pthread 등 거의 모든 실용 시스템이 채택했다.

Java에서 `synchronized` + `wait()`/`notify()`는 Mesa Monitor의 구현이다. 따라서 `while (!condition) { lock.wait(); }` 패턴이 표준이며, `notify()`가 특정 스레드를 지정할 수 없고 `notifyAll()`(broadcast)이 자연스럽게 동작하는 것도 Mesa 의미론의 특성이다. Java 객체의 `private` 필드는 공유 데이터, `synchronized` 메서드는 모니터 프로시저, `Object.wait()`/`notify()`/`notifyAll()`은 조건 변수 연산에 대응한다.

## Key Takeaways

- 모니터는 세마포어의 실수 가능성을 줄이기 위해 상호 배제를 언어/런타임 수준에서 자동 보장하는 고수준 추상화다
- 모니터의 세 구성 요소: 공유 데이터(private 필드), 프로시저(synchronized 메서드), 조건 변수(wait/signal)
- Hoare Monitor: signal 시 즉시 양보 → 조건 보장 → `if` 충분. 큐 3개 필요. 이론/교육용
- Mesa Monitor: signal 후 계속 실행 → 조건 재확인 필요 → `while` 필수. 큐 2개. 실용 시스템 표준
- Mesa에서 `while`이 필수인 이유: signal과 깨어남 사이에 다른 스레드가 상태를 변경할 수 있음 (spurious wakeup과 유사)
- Java `synchronized`는 Mesa Monitor — `while` 안의 `wait()`이 표준 패턴
- `notifyAll()`은 Mesa에서만 자연스럽게 동작: 깨어난 스레드들이 각자 `while`로 조건 재확인

## Quotes & References

> **핵심 규칙**: 모니터 내부에는 한 시점에 최대 하나의 스레드만 실행될 수 있다. 이것이 컴파일러/런타임에 의해 자동으로 보장된다.

> **한 줄 요약**: 모니터는 "공유 데이터와 그 데이터에 접근하는 코드를 하나로 묶고, 한 번에 하나의 스레드만 그 안에서 실행되도록 보장하는 동기화 추상화"이며, Java `synchronized`의 이론적 기반이다.

| 항목 | Hoare Monitor | Mesa Monitor |
|------|---------------|--------------|
| signal 후 동작 | 호출자가 즉시 양보 | 호출자가 계속 실행 |
| 조건 체크 | `if` 가능 | `while` 필수 |
| 내부 큐 | 3개 | 2개 |
| 채택 사례 | 이론/교육용 | Java, C#, POSIX 등 실용 시스템 |
