# Ingest: 미처리 원본 자료를 위키에 수집

`raw/` 폴더에서 아직 처리되지 않은 파일을 찾아 CLAUDE.md의 Ingest 워크플로우에 따라 위키에 통합한다.

---

## Phase 1: 미처리 파일 탐색

1. `raw/.processed` 파일을 읽어 이미 처리 완료된 파일 경로 목록을 가져온다.
2. `raw/` 하위의 모든 파일을 재귀 탐색한다.
   - **제외 대상**: `raw/.processed`, `raw/assets/*`, `.DS_Store`, `.gitkeep`
   - **대상 확장자**: `.md`, `.txt`, `.pdf`, `.html` 등 텍스트 기반 자료
3. `.processed` 목록에 없는 파일이 미처리 자료다.
4. 미처리 파일이 없으면 `"새로운 자료 없음. raw/ 폴더에 파일을 추가한 후 다시 실행하세요."` 출력 후 종료.
5. 미처리 파일이 있으면 목록을 출력하고 Phase 2로 진행한다.

---

## Phase 2: 각 파일에 대해 Ingest 실행

미처리 파일을 하나씩 순서대로 처리한다. 파일 하나당 아래 전체 과정을 수행한다.

### Step 1: 자료 읽기 및 분석

- 파일 전체를 읽는다.
- 다음을 파악한다:
  - **자료 유형**: 기사, 논문, 독서 노트, 회의록, 메모 등
  - **핵심 주제**: 이 자료가 다루는 중심 주제 1~3개
  - **등장 개체**: 인물, 조직, 도구, 장소 등 고유명사
  - **등장 개념**: 기술, 이론, 방법론 등 추상적 주제
  - **저자/출처 정보**: 있으면 추출
  - **날짜**: 자료의 작성일 또는 파일명에서 추출. 없으면 오늘 날짜 사용.

### Step 2: 요약 페이지 작성

`wiki/summaries/`에 새 요약 페이지를 생성한다.

- **파일명**: `YYYY-MM-DD-slug.md` (slug는 자료 제목을 kebab-case로 변환)
- **템플릿** (`templates/summary.md`) 구조를 따른다:
  ```yaml
  ---
  title: 자료 제목
  type: summary
  tags: [도메인 태그, 유형 태그]
  sources: [raw/ 내 파일 경로]
  created: YYYY-MM-DD
  updated: YYYY-MM-DD
  ---
  ```
- **Summary 섹션**: 자료의 핵심 내용을 3~5문단으로 요약. 원문의 논리 구조를 유지한다.
- **Key Takeaways 섹션**: 가장 중요한 인사이트 3~7개를 bullet point로.
- **Quotes & References 섹션**: 핵심 인용문이나 데이터 포인트 포함.
- **Related 섹션**: 이후 Step 3에서 생성/업데이트한 페이지 링크를 위키링크로 추가.

### Step 3: 개체/개념 페이지 생성 또는 업데이트

Step 1에서 파악한 개체와 개념 각각에 대해:

#### 기존 페이지가 있는 경우 (업데이트)
- `wiki/entities/` 또는 `wiki/concepts/`에서 해당 페이지를 찾는다.
- 새 자료에서 얻은 정보를 적절한 섹션에 추가한다.
- 프론트매터의 `sources` 배열에 원본 자료 경로를 추가한다.
- `updated` 날짜를 오늘로 갱신한다.
- 기존 내용과 **모순**되는 정보가 있으면 `> ⚠️ 확인 필요: [기존 내용]과 [새 정보]가 충돌합니다.` 형태로 표시한다.

#### 기존 페이지가 없는 경우 (생성)
- 해당 개체/개념이 위키에 독립 페이지를 가질 만큼 중요한지 판단한다.
  - 자료에서 단순 언급만 된 경우 → 페이지 생성 불필요, 요약 페이지 내에서만 언급.
  - 핵심 주제이거나 여러 자료에서 반복 등장할 가능성이 높은 경우 → 페이지 생성.
- 생성 시 `templates/entity.md` 또는 `templates/concept.md` 구조를 따른다.
- 파일명은 `kebab-case.md`.

### Step 4: 교차참조 추가

- 새로 만든 요약 페이지와 개체/개념 페이지 사이에 Obsidian 위키링크를 추가한다.
- 형식: `[[entities/파일명|표시이름]]` 또는 `[[concepts/파일명|표시이름]]`
- 기존 위키 페이지 중 새 자료와 관련 있는 페이지가 있으면 그 페이지의 Related 섹션에도 링크를 추가한다.

### Step 5: 인덱스 갱신

`wiki/index.md`를 열어 새로 만든 페이지들을 해당 범주 아래에 추가한다.

- 형식: `- [[경로|제목]] — 한 줄 요약`
- 범주별 알파벳 순으로 삽입.

### Step 6: 로그 기록

`wiki/log.md` 맨 아래에 append한다:

```markdown
## [YYYY-MM-DD] ingest | 자료 제목
- 원본: raw/경로/파일명
- 요약: 핵심 내용 한 줄
- 새 페이지: [[summaries/...]], [[entities/...]], [[concepts/...]]
- 업데이트: [[기존 페이지1]], [[기존 페이지2]]
```

### Step 7: .processed에 등록

처리 완료된 파일의 상대 경로를 `raw/.processed`에 한 줄 추가한다.

---

## Phase 3: 완료 보고

모든 미처리 파일 처리가 끝나면 아래 형식으로 요약을 출력한다:

```
## Ingest 완료

- 처리한 자료: N개
- 새로 만든 페이지: (목록)
- 업데이트한 페이지: (목록)
- 발견한 연결점/인사이트: (있으면 기술)
```

---

## Phase 4: Git 커밋

변경된 모든 파일을 staging하고 커밋한다.

- 커밋 메시지: `ingest: <N>개 자료 수집 (<자료 제목 나열>)`
- 자료 제목이 길면 첫 번째 자료만 명시하고 `외 N건`으로 축약.
